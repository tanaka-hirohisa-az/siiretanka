# 仕入単価推移レポート 仕様書

作成日：2026-07-08

## 1. 全体概要

### 1.1 目的

「仕入単価推移」レポートは、**納入品目**（お客様に納める完成品）と**材料**（その製造に使う原材料）の
2系統について、仕入単価の推移を月次で把握し、値上げ（または値下げ）を早期に検知し、その金額的な
影響を見積もることを目的とするレポートである。材料系のタブ群は、見積もった影響額を売価への転嫁
交渉の根拠として使うことも想定している。

### 1.2 データソース

Fabricデータウェアハウス（`dw_bee_genka-00001`）を単一のソースとする。レポート内の主な生テーブル
（インポート）は以下の通り。

| テーブル | 内容 |
|---|---|
| `TB_納入品目仕入先単価情報` | 納入品目×仕入先ごとの契約単価と適用開始日 |
| `TB_材料仕入先単価情報` | 材料×仕入先ごとの契約単価と適用開始日 |
| `TB_仕入実績情報` | 材料の実際の仕入実績（仕入日・数量・単価） |
| `TB_統合出荷情報` | 納入品目の出荷実績（稼働量算出に使用） |
| `TB_材料入出庫実績情報` | 材料の入出庫実績（稼働量算出に使用） |
| `納入品目情報` / `材料情報` | 納入品目／材料のマスタ情報 |
| `品目分類情報` | 分類マスタ（納入品目・材料で共有） |
| `tbl_calendar` | 日付ディメンション |

これらの生テーブルから、Power BI側の計算テーブル（DAXで生成される中間テーブル）が作られ、各タブの
ビジュアルはその計算テーブルを参照する。計算テーブルの生成ロジックはタブごとに異なり、これが
タブ間の数値の違いを生む最大の要因である（詳細は各タブの節を参照）。

### 1.3 タブ構成

| ページID | タブ名 | 対象 | 算出方式の系統 | 位置づけ |
|---|---|---|---|---|
| `b0c53a4cbd41c0616874` | 納入品目 | 納入品目 | 改定イベント単位 | 現行（この系統では唯一のタブ） |
| `2b6e1f3d7511a233f0ce` | 旧_材料 | 材料 | 改定イベント単位（旧方式） | **非推奨・参照用**。数値の正は下記2タブ |

| `47eede92d53b453450a9` | ページ 1 | ― | ― | 空ページ（未使用） |

「改定イベント単位」と「月次ベース」の違いは重要である。**改定イベント単位**（納入品目・旧_材料）は、
実際に契約単価が改定された日だけに1行が生成される方式で、月ごとの連続した水準の推移を追うことは
できない。**月次ベース**（材料の3タブ）は、材料×月の組み合わせで必ず（または条件付きで）1行を
持ち、改定がない月も含めて連続した推移を月次グラフで追える。

### 1.4 共通の仕組み

- **スライサー同期**：年・年月（階層）スライサーは、`tbl_calendar`の`年_yyyy年`/`年月_yy年MM月`を
  ソースとし、`syncGroup.groupName = "年月_yy年MM月"`によって**全タブ共通**で同期する。
  材料の3タブ（加重平均単価・契約単価・値上げ影響額）はさらに、分類名・在庫材料番号・在庫場所の
  スライサーも同期グループ（それぞれ`分類名`・`在庫材料番号`・`在庫場所`）で相互に連動する。
  一方、納入品目タブ・旧_材料タブの品目番号／仕入先／分類名スライサーには同期グループが設定
  されておらず、**そのページ内でのみ**有効（他タブと連動しない）。
- **年間予測影響額（影響額）の考え方**：どのタブでも「値上げ額（またはそれに相当する単価差）
  × 直近12ヶ月の実績数量」という共通の考え方で算出する。ただし「値上げ額」の算出方法（改定
  イベントの前後比較か、月次のローリング平均・契約単価の前月比較か）はタブごとに異なるため、
  タブをまたいで数値を単純比較しないこと。
- **タイトル表示**：材料の3タブは全チャートにカスタムタイトルを設定済み。納入品目・旧_材料
  タブは、カード（「選択期間中の値上げ額の合計」）以外のほとんどのビジュアルにカスタム
  タイトルが設定されておらず、Power BIの自動生成タイトル（列名を連結した表記）がそのまま
  表示される。

## 2. タブ別概要（利用想定）

### 2.1 納入品目タブ

お客様への納入品目について、仕入先との契約単価が改定されたタイミングを捉えるタブ。実際に
出荷があったかどうかとは関係なく、契約単価が変わった「その瞬間」の前後比較を一覧・グラフ化
する。用途：納入品目の値上げ改定の発生状況と、その1件ごとの影響額見積りの把握。

### 2.2 旧_材料タブ（非推奨）

材料版の「改定イベント単位」タブ。現在は**参照用としてのみ残置**されており、数値の正は
材料(加重平均単価)・材料(契約単価)の2タブである。新しいレポートを見る人は基本的に参照
不要。

### 2.3 材料(加重平均単価)タブ

材料の実際の仕入実績から算出した「実際に払っている単価」の月次推移を見るタブ。仕入があった
月だけで構成される。用途：実態としての値上がり状況の把握（実績指標）。

### 2.4 材料(契約単価)タブ

仕入先との契約単価そのものの月次推移を見るタブ。実際の仕入の有無とは関係なく、契約改定を
毎月拾う。用途：先行指標としての建値の動きの把握。

### 2.5 材料_値上げ影響額タブ

前年度末（前年12月）時点の実績単価を基準として固定し、当年の各月についてその基準からどれだけ
値上がり／値下がりしたか、および金額換算した影響額を、月別・分類別・材料別に見るタブ。用途：
値上げの売価転嫁交渉のための金額根拠資料。

## 3. タブ別詳細

### 3.1 納入品目タブ（ページID: `b0c53a4cbd41c0616874`）

#### データソースと生成ロジック

- 起点：`TB_納入品目仕入先単価情報`（納入品目×仕入先ごとの契約単価・適用開始日の生データ）
- `納入品目_仕入単価履歴`（計算テーブル）：上記を正規化し、品目×仕入先ごとに「前回単価」
  （直前に登録されていた契約単価）と「値上げ額」（今回単価－前回単価）を算出した履歴
- `納入品目_値上げイベント`（計算テーブル、**このタブの主データ**）：履歴のうち
  値上げ額が0でない行（＝実際に単価が変わった行）だけを残したもの。1行＝1回の改定
  イベント（品目×仕入先×改定日）
- `納入品目_月別純増減`（計算テーブル）：同一月内に複数回改定があった場合の、月初時点
  単価から月内最終単価までの正味の増減をまとめたもの（明細表の参考列にのみ使用）
- `納入品目_年平均稼働量`（計算テーブル）：`TB_統合出荷情報`から算出した直近12ヶ月・
  過去5年平均の出荷数量（影響額算出の数量根拠）

#### 主要項目定義

| 項目 | 定義 |
|---|---|
| 前回単価 | 同一品目×仕入先について、直前に登録されていた契約単価 |
| 値上げ額 | 今回の契約単価－前回単価（改定イベント1件ごとの単発の差分） |
| 値上げ率 | 値上げ額 ÷ 前回単価 |
| 改定方向 | 値上げ額>0なら「値上げ」、それ以外は「値下げ」 |
| 外れ値フラグ | 値上げ額の絶対値が5万円以上の改定に1を立てる（統計的な外れ値ではなく固定閾値） |
| 直近12か月稼働量 | 直近12ヶ月の実績出荷数量（`TB_統合出荷情報`ベース） |
| 年間予測影響額 | 値上げ額 × 直近12か月稼働量（この改定が今後1年続いた場合の影響額の見積り。実際に発生した累計コストではなく、単発の単価差を年間数量に掛けた予測値） |
| 年間予測影響額_値上げのみ／値下げのみ | 上記を値上げ・値下げで分けたもの |
| 月別_純増減額合計（明細のみ） | 同一月・同一品目内で複数回改定があった場合、月初時点単価から月内最終単価までの正味の増減額（`納入品目_月別純増減`から参照） |

### 3.3 材料(加重平均単価)タブ（ページID: `55cba7d188481af95d14`）

#### データソースと生成ロジック

計算テーブル`材料_加重平均単価_月次`（`TB_仕入実績情報`から生成）。材料×月ごとに、
当該月を含む直近6ヶ月の実仕入を合算して単価をならした「加重平均単価」を算出する。行が
存在するのは、その6ヶ月の間に実際の仕入が1件以上あった月のみ。

#### 主要項目定義

| 項目 | 定義 |
|---|---|
| 加重平均単価 | 当該月を含む直近6ヶ月の仕入金額合計 ÷ 仕入数量合計（1回の取引の単価ではなく、直近6ヶ月をならした値） |
| 前回加重平均単価 | 前月の加重平均単価。ただし前月を含む直近6ヶ月に仕入が1件もない場合は、直近で実際に仕入があった月の単月平均（その月の仕入金額÷仕入数）で代用 |
| 値上げ額 | 加重平均単価 － 前回加重平均単価 |
| 値上げ率 | 値上げ額 ÷ 前回加重平均単価 |
| 月内仕入数／月内仕入金額 | その月に実際にあった仕入のみの数量・金額（ローリング加算を含まない単月の実績値。仕入がない月はこの2列が空欄になる） |
| 直近12か月稼働量 | 直近12ヶ月の実績仕入数量の合計 |
| 年間予測影響額（＋値上げのみ／値下げのみ） | 値上げ額 × 直近12か月稼働量（この改定が1年続いた場合の影響額の見積り） |

### 3.4 材料(契約単価)タブ（ページID: `c3d4e5f6a7b8901234cd`）

#### データソースと生成ロジック

計算テーブル`材料_契約単価_月次`（`TB_材料仕入先単価情報`から生成）。材料×仕入先ごとに
登録された契約単価と適用開始日から、材料×仕入先×月の「その月時点で有効な契約単価」を
求め、材料単位で仕入先間の単純平均をとったものが「契約単価」。行はその材料の最初の契約
単価登録月から前月まで毎月存在する（仕入の有無とは無関係）。

#### 主要項目定義

| 項目 | 定義 |
|---|---|
| 契約単価 | 当該月時点で有効な、各仕入先の契約単価の単純平均 |
| 契約単価_前月契約単価 | 前月の契約単価。前月の行が存在しない場合は空欄（材料(加重平均単価)タブと異なり、それより前へのフォールバック検索は行わない） |
| 契約単価_値上げ額 | 契約単価 － 契約単価_前月契約単価 |
| 契約単価_値上げ率 | 契約単価_値上げ額 ÷ 契約単価_前月契約単価 |
| 契約単価_直近12か月稼働量 | 直近12ヶ月の実績仕入数量（`TB_仕入実績情報`から集計。契約の有無とは無関係に実際の仕入数量を使用） |

### 3.5 材料_値上げ影響額タブ（ページID: `a1b2c3d4e5f6789012ab`）

#### データソースと生成ロジック

材料(加重平均単価)タブと同じ計算テーブル`材料_加重平均単価_月次`に、追加の計算列のみで
実現している（新規テーブル・リレーションなし）。

#### 主要項目定義

| 項目 | 定義 |
|---|---|
| 年度基準単価 | 前年12月時点の加重平均単価（前回加重平均単価と同じフォールバック規則：前年12月を含む直近6ヶ月に仕入が1件もなければ、直近の実仕入月の単月平均で代用）。その材料・その年度を通じて固定値 |
| 年初来値上げ額 | 当月の加重平均単価 － 年度基準単価 |
| 年初来影響額 | 年初来値上げ額 × 直近12か月稼働量 |
| 年初来影響額_値上げのみ／値下げのみ | 上記を値上げ・値下げで分けたもの |
| 年初来値上げ率 | 年初来値上げ額 ÷ 年度基準単価 |
